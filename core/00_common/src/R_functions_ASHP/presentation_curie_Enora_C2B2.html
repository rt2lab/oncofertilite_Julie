autorise les listes, les matricescomme valeur de cellule as_tibble()

*la structure de la variable est donne entre parenthses sous chaque nom de variable

*des informations concernant le jeu de donnes, notamment ses dimensions (Source: local data frame), sont affiches dans la partie suprieure de laffichage

*seules les variables qui peuvent tre entirement affiches dans la fentre sont visibles

*ET seules les premires lignes sont affiches.

Si on veut voir plus :

o options(tibble.print_max = n, tibble.print_min = m): if there are more than nrows, print only the first m rows. Use options(tibble.print_max = Inf) to always show all rows.

o options(tibble.width = Inf) will always print all columns, regardless of the width of the screen.

Dplyr

select() : selectionner des colonnes

Select (d, var1, var2,) : permet de selectionner des colonnes

curie[,c("rtdeb","rtfin")]
# est quivalent  :
select(curie, rtdeb,rtfin) ##pas besoin de ""
select(curie, hormo:finhor)##selection de plusieurs colonnes contigues avec ":"
select(curie, -BMI, -age) ##anti selection par nom de variables
select(curie, -starts_with("neobio"))
select(curie, starts_with("adj", ignore.case=TRUE))
select(curie, ends_with("her"))
select(curie, -contains("q1"))
select(curie, matches("uicc"))

filter() : filtrer les lignes

Permet de filtrer les lignes du jeu de donnes qui satisfont un critre. Son criture est trs proche de la fonction subset :

filter(curie, datchir > "2010-01-01")

# avec plusieurs critres : 
#ces 2 critures sont identiques : 
filter(curie, groupe =="luminal" & datchir > "2010-01-01")
filter(curie, groupe =="luminal", datchir > "2010-01-01")

# le symbole | s'utilise comme en base : 
filter(curie, groupe =="luminal" | groupe =="her2+")

arrange()

Trie le jeu de donnes, dans lordre croissant ou dcroissant selon un ou plusieurs critres. Les donnes manquantes sont relgues en fin de tableau.

arrange(curie, datexam) ##ordre croissant par defaut
arrange(curie, desc(datexam)) ##par ordre decroissant

summarise()

Le rsultat dune fonction summarise est un tbl_df autant de colonnes que de variables calcules.

Enchaner les oprations de manipulation des donnes et faire les calculs par groupes :

pipe operator %>% ( {magrittr})

Evite limbrication des fonctions les unes dans les autres

%>% peut tre lu comme ensuite ou puis .

Loprateur %>% passe comme premier argument la fonction quon trouve immdiatement sa droite, ce qui se trouve sa gauche.

Si on souhaite passer droite du pipe un argument qui nest pas en premire position dans la fonction que lon souhaite utiliser, on utilise .

group_by()

Dcoupe un jeu de donnes pour raliser des oprations sur chacun des sous-ensembles.

Dans un pipe, ds lors quun group_by est introduit, toutes les oprations qui suivent sont ralises pour chacun des sous-ensembles de la base.

Group_by (data, var) : groupe les data par var.
