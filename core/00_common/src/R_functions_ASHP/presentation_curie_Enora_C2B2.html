autorise les listes, les matrices…comme valeur de cellule as_tibble()

*la structure de la variable est donnée entre parenthèses sous chaque nom de variable

*des informations concernant le jeu de données, notamment ses dimensions (Source: local data frame), sont affichées dans la partie supérieure de l’affichage

*seules les variables qui peuvent être entièrement affichées dans la fenêtre sont visibles

*ET seules les premières lignes sont affichées.

Si on veut voir plus :

o options(tibble.print_max = n, tibble.print_min = m): if there are more than nrows, print only the first m rows. Use options(tibble.print_max = Inf) to always show all rows.

o options(tibble.width = Inf) will always print all columns, regardless of the width of the screen.

Dplyr

select() : selectionner des colonnes

Select (d, var1, var2,…) : permet de selectionner des colonnes

curie[,c("rtdeb","rtfin")]
# est équivalent à :
select(curie, rtdeb,rtfin) ##pas besoin de ""
select(curie, hormo:finhor)##selection de plusieurs colonnes contiguëes avec ":"
select(curie, -BMI, -age) ##anti selection par nom de variables
select(curie, -starts_with("neobio"))
select(curie, starts_with("adj", ignore.case=TRUE))
select(curie, ends_with("her"))
select(curie, -contains("q1"))
select(curie, matches("uicc"))

filter() : filtrer les lignes

Permet de filtrer les lignes du jeu de données qui satisfont un critère. Son écriture est très proche de la fonction subset :

filter(curie, datchir > "2010-01-01")

# avec plusieurs critères : 
#ces 2 écritures sont identiques : 
filter(curie, groupe =="luminal" & datchir > "2010-01-01")
filter(curie, groupe =="luminal", datchir > "2010-01-01")

# le symbole | s'utilise comme en base : 
filter(curie, groupe =="luminal" | groupe =="her2+")

arrange()

Trie le jeu de données, dans l’ordre croissant ou décroissant selon un ou plusieurs critères. Les données manquantes sont reléguées en fin de tableau.

arrange(curie, datexam) ##ordre croissant par defaut
arrange(curie, desc(datexam)) ##par ordre decroissant

mutate()

Permet la transformation d’une variable existante ou la création d’une nouvelle variable dans le jeu de données.

mutate(data, nouvelle_var = …)

mutate = fonction dite “window” : l’opération qui est réalisée sur la variable restitue une variable de même taille. C’est une transformation

summarise()

summarize (ou summarise) = fonction dite de “résumé”. À l’inverse de mutate, quand une fonction summarize est appelée, elle retourne une seule information (ex: moyenne, variance, effectif… )

Le résultat d’une fonction summarise est un tbl_df à autant de colonnes que de variables calculées.

summarise(curie, gg_moy=mean(ggenv, na.rm=T))
##     gg_moy
## 1 1.432749
#calcul de plusieurs indicateurs à la fois avec ","
summarise(curie, 
          gg_moy=mean(ggenv, na.rm=T),
          gg_et = sd(ggenv, na.rm=T))
##     gg_moy   gg_et
## 1 1.432749 2.54423

Enchaîner les opérations de manipulation des données et faire les calculs par groupes :

pipe operator %>% ( {magrittr})

Evite l’imbrication des fonctions les unes dans les autres

%>% peut être lu comme « ensuite » ou « puis ».

L’opérateur %>% passe comme premier argument à la fonction qu’on trouve immédiatement à sa droite, ce qui se trouve à sa gauche.

Si on souhaite passer à droite du pipe un argument qui n’est pas en première position dans la fonction que l’on souhaite utiliser, on utilise “.”

group_by()

Découpe un jeu de données pour réaliser des opérations sur chacun des sous-ensembles.

Dans un pipe, dès lors qu’un group_by est introduit, toutes les opérations qui suivent sont réalisées pour chacun des sous-ensembles de la base.

Group_by (data, var) : groupe les data par var.

gather()

“reshaping”

passage du format “wide” (colonne les unes à côté des autres)

au format long (colonnes les unes après les autres)

Gather (d, key , value, var1, var2,…)

Dans la colonne « key », il y aura le nom des anciennes colonnes reproduit autant de fois que voulu

Dans la colonne “value”, il y aura la valeur correspondante

La fonction inverse est spread()
